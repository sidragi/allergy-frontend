How We Built AllergyDetector

A step-by-step look at our technical implementation

1

Research Phase

Identifying Allergy Types

We began by researching common allergy types to determine which would benefit most from AI analysis. After evaluating several options, we focused on:

  • Pollen allergies - Seasonal and highly location-dependent
  • Skin allergies - Visible symptoms suitable for image analysis
2

Data Collection

Building Our Dataset

We created a binary classification dataset with two categories:

Healthy Skin

Sourced from Kaggle public datasets containing images of normal, healthy skin

Allergic Reactions

Collected from Dermnet, featuring various allergic skin conditions

3

Model Deployment

API Development

We saved the trained model as a .h5 file and created a REST API endpoint using FastAPI: 

from fastapi import FastAPI, UploadFile, File, HTTPException
from fastapi.middleware.cors import CORSMiddleware
import numpy as np
import cv2
import tensorflow as tf
import keras
import io
import requests
from pydantic import BaseModel

app = FastAPI()

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

model = keras.saving.load_model("allergyDetection.h5")

@app.get("/")
async def root():
    return {"message": "Allergy Detection API is up."}

class ImageURL(BaseModel):
    url: str

@app.post("/predict_url")
async def predict_url(data: ImageURL):
try:
    response = requests.get(data.url)
    if response.status_code != 200:
        raise HTTPException(status_code=400, detail="Failed to fetch image from URL")

     image_bytes = np.asarray(bytearray(response.content), dtype=np.uint8)
                      img = cv2.imdecode(image_bytes, cv2.IMREAD_COLOR)
              
                      if img is None:
                          raise HTTPException(status_code=400, detail="Image decoding failed")
              
                      resize = tf.image.resize(img, (256, 256))
                      sev = float(model.predict(np.expand_dims(resize / 255.0, 0))[0][0])
              
                      if sev < 0.1:
                          result = "Severe allergy. Please visit a doctor immediately."
                      elif sev < 0.3:
                          result = "Moderate allergy. Please consult a doctor."
                      elif sev < 0.5:
                          result = "Mild allergy. Monitor the condition."
                      elif sev < 0.7:
                          result = "Low chance of allergy. Keep an eye on symptoms."
                      else:
                          result = "No signs of allergy."
              
                      percentage = round((1 - sev) * 100, 2)
              
                      return {"result": result, "severity": float(1 - sev), "severity percentage": percentage}
              
                  except Exception as e:
                      raise HTTPException(status_code=500, detail=f"URL Prediction failed: {str(e)}")

The API was deployed to Render for reliable cloud hosting with automatic scaling.

5

Pollen Integration

Location-Based Data

For pollen allergy detection, we integrated two key APIs:

Google Geolocation API

Gets the user's current location to provide localized pollen data

Pollen API

Provides real-time pollen levels and forecasts for the user's area

6

Chatbot Integration

Unifying the Experience

We brought all components together using Botpress to create a seamless conversational interface:

  • Integrated both skin analysis and pollen APIs
  • Designed conversational flows for different allergy types
  • Implemented natural language understanding for symptom queries
  • Added personalized recommendation generation

The chatbot serves as the central interface that orchestrates all our services, providing users with a unified allergy management experience.
